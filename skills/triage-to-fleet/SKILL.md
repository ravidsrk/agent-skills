# Triage-to-Fleet

Coordinator for Matt’s triage state machine with **parallel verification**.



## ⚠️ HARD BASE: Orca `orchestration`

**This skill is built on Orca orchestration — not on other skills in this pack, and not on in-process subagents.**

| Layer | Owns | Source |
|-------|------|--------|
| **Runtime** | tasks, dispatch, `worker_done`, gates, DAG, worktrees | Orca (`orca orchestration …`) |
| **Grammar** | CLI + lifecycle rules | **`orchestration` skill from the Orca CLI** (not this repo) |
| **This skill** | *what / when / why* on top of that grammar | this repo |
| **Workers** | AFK playbooks (Matt `/implement`, `/tdd`, …) | mattpocock/skills or this pack |

**Preflight (stop if any fail):** `orca status --json` running · orchestration experimental on · `orchestration` skill loaded · never substitute Task/subagent tools for `task-create` + `dispatch`.

**Full handoff** ("give this to another agent") → `orca-cli`, not supervised `dispatch --inject`, unless the user asked to supervise / wait for `worker_done`.

## Important

- Triage is for issues **you didn’t create** (incoming bugs/requests).  
  **Do not triage** `/to-tickets` output.
- Every tracker comment starts with:  
  `> *This was generated by AI during triage.*`

## Phase graph

```
ORIENT (labels map from setup-matt-pocock-skills)
  → BUCKET (unlabeled / needs-triage / needs-info with activity)
  → VERIFY WAVE (AFK workers per issue: reproduce, redundancy, out-of-scope KB)
  → RECOMMEND + HUMAN GATE (category + state)
  → APPLY (brief / needs-info / wontfix / ready-for-human)
  → optional IMPLEMENT FLEET on ready-for-agent (or hand to ready-agent-drain / matt-ship slice)
```

## Verify worker prompt (per issue)

```text
Triage verify only for issue <id>. Do not change labels.
1) Read issue + comments
2) Codebase redundancy search by domain concept
3) Check .out-of-scope/
4) Attempt reproduce (bug) or claim-check (enhancement)
5) Report: confirmed | failed | insufficient-info + evidence paths
worker_done with payload findings. Do NOT post to tracker.
```

Coordinator posts tracker comments and applies roles **only after human direction** (except pure read-only listing).

## Ready-for-agent

When state is `ready-for-agent` and brief posted:

- Either enqueue **`ready-agent-drain`**, or  
- `task-create` implement workers for a bounded set with human cap.
